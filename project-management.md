## Wiki文档

### 1. 系统设计说明

#### 系统功能

- **用户认证**：支持登录功能，区分管理员和普通用户
- **文章管理**：支持创建、查看、编辑文章
- **评论系统**：支持对文章发表评论
- **分类和标签**：文章支持分类和标签管理
- **响应式设计**：适配不同屏幕尺寸

#### 技术选型

- **后端**：Node.js + Express + CORS
- **前端**：HTML5 + CSS3 + JavaScript
- **数据存储**：JSON文件
- **部署**：可部署到任意支持Node.js的云平台

#### UML图的使用

1. **用例图**：
   - 参与者：博主、访客、管理员
   - 用例：发布文章、编辑文章、查看文章、评论、登录、管理用户
   - 关系：登录是发布文章和编辑文章的前置条件（include关系）

2. **类图**：
   - 实体类：User、Post、Comment、Category、Tag
   - 关联关系：
     - User 1:N Post
     - User 1:N Comment
     - Post 1:N Comment
     - Post N:N Category
     - Post N:N Tag

3. **状态图**：
   - 关键对象：文章
   - 状态：草稿、待审核、已发布、已归档
   - 触发事件：创建文章、提交审核、审核通过、审核拒绝、归档文章

### 2. 开发记录

#### 开发中遇到的问题及解决方案

1. **问题**：跨域请求问题
   - **解决方案**：使用CORS中间件，允许前端跨域请求

2. **问题**：数据存储方案选择
   - **解决方案**：使用JSON文件存储数据，简化开发和部署

3. **问题**：用户认证安全性
   - **解决方案**：实现简单的登录验证，后续可考虑使用JWT

4. **问题**：前端页面响应式设计
   - **解决方案**：使用CSS媒体查询，适配不同屏幕尺寸

5. **问题**：分类管理功能的实现
   - **解决方案**：在后端添加分类管理API，前端添加分类选择和管理界面

#### 技术栈学习

- **Node.js**：学习了基本的Node.js语法和Express框架的使用
- **Express**：学习了如何创建API接口和处理HTTP请求
- **前端开发**：学习了HTML5、CSS3和JavaScript的基本用法
- **版本控制**：学习了Git的基本操作和分支管理
